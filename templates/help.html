{% if active_topic == "getting-started" %} Getting Started

First launch checklist

What PrepaC does

PrepaC helps you prepare source media, pack releases, post them, optionally share generated NZBs to Newznab-compatible destinations, and then clean up processed content.

  1. Create the first admin account during the setup flow.
  2. Open Settings and configure your paths before running jobs.
  3. Configure one or more posting providers if you plan to use the Posting screen.
  4. Configure Share destinations if you want uploads sent to Newznab-compatible indexers after posting or by manual import.
  5. Start with a small job before processing larger batches.

Persistent data

Keep /config persistent across upgrades so your settings and history remain available.

Workflow

Most setups will follow Settings → Prepare → Packing → Posting → Share → Clean.

Self-hosted compatibility remains supported for HTTP-only local/LAN setups, with optional hardening controls available in Settings and environment variables.

{% elif active_topic == "prepare" %} Prepare

Create working jobs from TV seasons or movie folders

The Prepare screen scans your configured TV and movie roots, lets you choose content, and creates working jobs for later Packing.

  • TV mode lets you search shows and select one or more seasons.
  • Movie mode lets you search movie folders and build jobs from the selected results.
  • The queue preview shows detected bracket information, target naming, and destination paths before the copy starts.
  • Prepared jobs become the source for Packing.
{% elif active_topic == "packing" %} Packing

Turn prepared jobs into release output

The Packing screen scans prepared jobs, estimates the output, and creates the packed release artifacts used later by Posting.

  • Packing writes release output to the configured packing output root.
  • RAR and PAR2 estimates are shown during scan.
  • The optional Freeimage key is only used for image upload related packing features and is not used for title metadata lookup.
{% elif active_topic == "posting" %} Posting

Post packed releases with your configured providers

The Posting screen scans packed jobs, starts provider-backed uploads, and shows live progress and transfer state.

  • Posting providers are managed from the dynamic provider builder in Settings.
  • Existing Provider 1 and Provider 2 setups carry forward automatically for current users.
  • For providers after Provider 1, use Prioritize jobs up to (GB) to prefer smaller jobs on specific providers first. Set it to 0 to keep that provider in the same default availability pool as Provider 1.
  • Posting tracks phase, transfer rate, transferred amount, percent complete, and ETA.
  • Completed posting output becomes eligible for Share.
  • Finished posting jobs move to Posting History.
{% elif active_topic == "share" %} Share

Submit posted or imported NZBs to Newznab-compatible destinations

The Share screen lets you review candidates, import RARred NZB + template bundles, preview destination categories, and submit to one or more configured destinations.

Candidate sources

Share can use successful Posting output or imported bundles made of a RARred NZB and template file. Imported bundles can also include an optional MediaInfo XML override.

Destination routing

PrepaC can auto-detect categories for TV and movies and let you override the final category before submission.

  • Single import handles one RARred NZB and one template at a time.
  • Mass import can pair many NZBs and templates using filename matching and template-content matching.
  • Share generates an NFO and metadata XML for submission when enabled by the destination.
  • Active Share jobs can be cancelled or removed directly from the Active Share Jobs card.
  • Failed Share jobs can be retried from the Share page.
  • Removed Share candidates stay hidden, but importing the same bundle again creates a fresh candidate.
  • Share History keeps a record of completed, failed, and cancelled Share jobs.
{% elif active_topic == "clean" %} Clean

Preview cleanup before deleting anything

The Clean screen uses app history and optional Plex watched-state information to suggest deletion candidates.

  • Dry Run shows what would be removed without deleting files.
  • Use Recycle Bin moves files into the recycle location instead of deleting directly.
  • When every season folder in a show is selected, the interface tells you that the show folder will also be removed.
{% elif active_topic == "settings" %} Settings

Configure paths, providers, destinations, and app behavior

The Settings page controls paths, posting providers, optional Plex integration, Share destinations, and runtime behavior.

Paths

Configure your main paths before using Prepare, Packing, Posting, Share, or Clean.

Share destinations

Share destinations support multiple targets, category overrides, and optional attachments.

  • Configure destination paths before using Prepare, Packing, Posting, or Share.
  • Use the provider builder to add, remove, clone, or reorder posting providers before starting uploads.
  • Use Prioritize jobs up to (GB) on providers after Provider 1 when you want smaller jobs to prefer specific providers first.
  • Refresh Share destination categories after adding or changing a destination.
  • Prepare permissions mode supports compatibility and hardening options: legacy_open (default), shared_safe, and owner_strict.
  • Session cookie behavior supports HTTP-only local/LAN and HTTPS deployments via configurable cookie mode.
  • Metrics can use token-based scraping when configured; otherwise metrics use authenticated access behavior.
  • Sign-in and password reset use temporary lockouts after repeated failed attempts.
