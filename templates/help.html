{% if active_topic == "getting-started" %} Getting Started

First launch checklist

Recommended order

Create your admin account, configure paths, optionally connect Plex, then configure posting providers before starting real jobs.

  1. Create the first admin account during the first-run setup flow.
  2. Open Settings and confirm source and destination paths.
  3. Set the default release tag if you want something other than PrepaC.
  4. Optionally connect Plex for poster lookups, watched-state cleanup, and library-aware behavior.
  5. Configure one or two posting providers before using the Posting screen.

What PrepaC does

PrepaC turns source media into workflow-managed jobs, packs those jobs into release artifacts, posts them, and later helps clean them up.

What is stored persistently

Application settings, authentication details, job history, and workflow state live in the SQLite database under /config.

{% elif active_topic == "prepare" %} Prepare

Build queue previews before creating jobs

The Prepare screen searches TV and movie source folders, lets you select seasons or movies, and shows a queue preview before any copy operation starts.

TV workflow

Search for a show, load available seasons, select one or more seasons, then build the queue preview.

Movie workflow

Search by movie folder name, select one or more results, then build the queue preview.

  • The queue preview shows the chosen bracket, destination folder, destination path, and detected tags.
  • If the end tag is left unchanged, PrepaC uses PrepaC by default.
  • Prepare jobs create the working folders that later appear in Packing.
{% elif active_topic == "packing" %} Packing

Turn prepared jobs into RAR and PAR2 output

The Packing screen scans the configured watch folder, estimates output, and starts packing jobs.

  • RAR and PAR2 estimates are shown during scan.
  • Packing writes output under the configured packing output folder.
  • The optional Freeimage key is only used for thumbnail upload during packing.
Important

The Freeimage API key is not a TMDB key and is not used for title metadata search.

{% elif active_topic == "posting" %} Posting

Use provider-backed Nyuu posting jobs

The Posting screen scans packed jobs, selects eligible releases, and starts uploads using your configured providers.

  • Provider 1 and Provider 2 can be enabled independently.
  • Posting tracks phase, progress, transfer rate, transferred amount, and ETA.
  • Completed jobs move into Posting History.
{% elif active_topic == "clean" %} Clean

Preview before deleting

The Clean screen combines Plex watched-state logic and previously prepared content to suggest deletion candidates.

  • Dry Run shows what would be removed without deleting files.
  • Use Recycle Bin moves files into the recycle location instead of deleting directly.
  • When every season folder in a show is selected, the interface states that the show folder will also be removed.
{% elif active_topic == "settings" %} Settings

Application configuration reference

The Settings page controls paths, Plex integration, packing behavior, posting provider details, and cleanup safety behavior.

Plex

Use the built-in Plex sign-in flow or manually provide a token and library names.

Posting providers

Configure hosts, ports, SSL, credentials, connections, and posting behavior from one place.

  • Secrets can come from the UI, environment variables, or secret files under /run/secrets.
  • The Freeimage key is optional and only used during packing.
{% elif active_topic == "operations" %} Operations

Running PrepaC in production

Gunicorn runtime

PrepaC now starts behind Gunicorn instead of Flask’s development server. The startup script sizes workers from host CPU count and stays overrideable through environment variables.

  • Default workers: (CPU cores × 2) + 1
  • Default threads scale by host size
  • Override with GUNICORN_WORKERS, GUNICORN_THREADS, GUNICORN_TIMEOUT, and related variables
  • Keep /config persistent for safe upgrades and rollback
