# Usage

## Prepare
Scan your TV or movie source folders and create working jobs.

## Packing
Build release output from prepared jobs.

## Posting
Start uploads from packed jobs using your configured providers. Current Provider 1 and Provider 2 setups carry forward into the dynamic provider builder. For providers after Provider 1, **Prioritize jobs up to (GB)** lets you prefer smaller jobs on specific providers first, while **0** keeps a provider in the same default pool as Provider 1.

## Share
Review share candidates, import bundles, and submit to your configured destinations. Active Share jobs can be cancelled or removed from the Share screen, and Share History keeps the finished record.

## Clean
Review deletion candidates before removing content.

## Sign-in behavior
Sign-in and password reset include temporary protection against repeated failed attempts.

## Self-hosted compatibility controls
Session cookie behavior supports both HTTP-only local/LAN setups and HTTPS deployments through configurable mode settings.
Metrics can use token-based scraping when a metrics token is configured; otherwise, metrics keep authenticated access behavior.
