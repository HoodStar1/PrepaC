
# Usage

- Prepare: scan and process files
- Packing: bundle processed items
- Posting: send jobs
- Clean: remove completed files

The interface guides you through each step.


## Provider priority

When the Provider 2 max job size when provider 1 is busy setting is above 0, Provider 1 prioritizes larger posting jobs first while Provider 2 can absorb smaller jobs. Set it to 0 to disable that behavior.


Active Prepare Jobs are shown on the Prepare screen. Cancelled, failed, and completed prepare jobs move to Prepare History. The duplicate completed packing summary panel was removed because Packing History already includes that information.
